Simple ToDo App

This project involved creating a ToDo app using React Native. The FlatList component provided by React Native was utilized for efficient rendering of tasks. SafeAreaView is also employed to ensure compatibility with different device layouts CheckBoxes, Input, and text elements from React Native Elements are utilized to implement a task management interface. Default tasks are displayed from a state array, each with properties like a unique identifier, completion status, and description. Tasks are styled with a strikethrough effect if completed, and an input box allows users to add new tasks to the list.

One challenge I faced was getting the check mark boxes to actually appear in color when selected.

I wanted a monochromatic look for this page. I felt that would keep it simple, allowing to user to focus on the main point of the app which is to create your todo list and complete the tasks.

Exercise App - First Stage

This project involved creating a simple exercise app with multiple screens. The home screen presents a menu of exercises that have been categorized and are displayed as styled buttons. Upon clicking a button, the corresponding exercise component is rendered, with the exercise name displayed at the top. Components are created to handle these exercises. The RepetitionExercise component allows users to increment a counter of repetitions and reset it, while the DurationExercise component enables users to start and reset a timer. The RepetitionDurationExercise component combines the increment counter and timer features.

One challenge was getting the timer feature to work properly. It took some time to set the code up in a way that allowed the timer to function in the way I wanted.

This project was the beginning concept I built on for a later project. For that reason, I spent minimal time on the design on the app. I wanted to ensure the app was functional first.

Login and Registration for ToDo App

This project involved building a ToDo app using React Native and implementing features such as a Login screen, Registration screen, and navigation between them using React Navigation. The Login screen consists of two input fields for username and password, with corresponding buttons for login and registration. Upon login, the app navigates to the ToDo screen if the credentials match the stored data. The Registration screen includes input fields for personal details like first name, last name, username, password, etc., with validation for each field based on specific patterns using Regular Expressions. Error messages are displayed dynamically based on input validation, guiding users to enter correct information.

One challenge I faced during this project involved making sure the information entered during the signup process was properly saved so it could be used later to login. After trying various versions of code I was finally able to find a solution to work.

I chose to keep the colors consitent across all pages. I used brighter colors to help the buttons stand out.

Exercise App

This project involved building a React Native application with React Native Elements, focusing on creating a Home component that includes a FlatList of buttons for navigating to different exercises. Each exercise type is represented by a screen. The Home component utilizes React Native Element buttons within the FlatList. The DurationExercise, RepetitionExercise and RepetitionDurationExercise components are registered as screens and use navigation params to display activity names, allow users to increase activity count or start timers, and include options to reset activities. Each exercise in the array includes a "Suggested" exercise button for direct navigation and a "Home" button for returning to the Home screen, ensuring smooth navigation flow within the application.

I struggled most with getting the "Suggested" button to work on all pages. It took a lot of time adjusting the code before I realized it was overly complicated. In order for the button to work, I just needed to keep the code very simple.

I wanted the list of exercise buttons on the home to appear bright and eyecatching. On the individual pages, I decided it was best to keep the design simple so it was clear to the user how to make the timer and increment features work.

Quiz App

This project involved the development of a React Native quiz application that supports three types of questions: True/False, Multiple Choice, and Multiple Answer. The app includes a Question component and a Summary component.

The Question component receives a list of questions and their corresponding answers as route parameters, allowing users to select their answers and navigate to the next question.

The Summary component displays all questions, user-selected answers, and the correct answers, calculating and displaying the total score. Navigation between questions and to the Summary screen is facilitated by React Native Navigation. The app's functionality ensures that users cannot navigate back to previous questions, and it accurately totals the user's score based on their answers.

One challenge I face was having the results properly show on the summary page. The multiple answer questions weren't highlighting the correct answers so I had to add additional code.

I chose to keep the majority of the automatic formatting as I wanted all screens to keep a simplistic look.
